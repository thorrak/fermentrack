Trigger SQLite Fix for Django 2.0+

In between the original version of Django that was used by Fermentrack and modern versions, certain backwards-incompatible changes were implemented that can cause issues with SQLite databases. These can be fixed programatically, but as we can't automaticaly detect when the version of Django changes we only run this command when manually triggered by the user.

This command is running now, and may take up to a minute to complete - depending on the size of your SQLite database. As soon as the command completes, Fermentrack will restart and you will be able to access it as you would normally.
